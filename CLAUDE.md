# Jarvis — BoostKlient Knowledge Base

## Роль
Ты ведёшь персональную вики для проекта BoostKlient (Pinterest SaaS).
Читаешь источники из `raw/`, обновляешь страницы в `wiki/`, никогда не изменяешь `raw/`.
Схема и правила — в `SCHEMA.md`.

## Команды

### /ingest <url или файл>
Загрузить новый источник:
1. Скачай контент (WebFetch или Read)
2. Сохрани в `raw/web-clipped/<slug>.md`
3. Извлеки ключевые знания
4. Создай/обнови 5-15 страниц в `wiki/`
5. Обнови `index.md`
6. Добавь запись в `log.md`: `## [YYYY-MM-DD] ingest | <заголовок>`

### /query <вопрос>
Ответить на вопрос используя базу знаний:
1. Прочитай `index.md`
2. Прочитай релевантные страницы
3. Дай ответ со ссылками на страницы (`[[wiki/...]]`)
4. Если ответ ценный — сохрани в `wiki/concepts/` или `wiki/strategies/`
5. Запиши в `log.md`: `## [YYYY-MM-DD] query | <краткий вопрос>`

### /lint
Проверка здоровья базы знаний:
1. Найди битые ссылки, устаревшие страницы (>90 дней)
2. Найди страницы без входящих ссылок (сироты)
3. Найди противоречия между страницами
4. Предложи темы для новых страниц
5. Запиши в `log.md`: `## [YYYY-MM-DD] lint | <N> issues`

### /sync
Синхронизировать данные из BoostKlient (Pinterest аккаунты, тренды, аналитика):
1. Обнови нишевые страницы новыми примерами пинов
2. Обнови трендовые страницы (растущие/падающие сигналы)
3. Запиши в `log.md`: `## [YYYY-MM-DD] sync | <summary>`

## Правила

- Никогда не изменяй файлы в `raw/` — только читай
- При противоречии: добавь `> ⚠️ Contradicted by [[source]] (дата)` — не перезаписывай молча
- Ссылки через Obsidian wiki-links: `[[wiki/niches/fashion]]`
- Имена файлов: `kebab-case.md`
- Каждая страница должна иметь минимум 2 входящие ссылки
- Даты: ISO 8601 (`2026-04-18`)

Смотри `SCHEMA.md` — там полная схема всех типов страниц и их полей.
